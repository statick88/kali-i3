# Customization

## Color Palette

The NEON MINIMAL theme uses a consistent palette across i3, SDDM, tmux, and zsh:

| Variable | Hex | Usage |
|----------|-----|-------|
| `NEON_BG` | `#0A0A10` | Background (tmux bar, i3 bar, SDDM theme) |
| `NEON_FG` | `#E0E0E0` | Primary text |
| `NEON_ACCENT` | `#008B8B` | Primary accent (teal) — used for active elements |
| `NEON_CYAN` | `#00BCD4` | Secondary accent — used for highlights |
| `NEON_GREEN` | `#00FF66` | Success indicators |
| `NEON_ALERT` | `#C71585` | Error/alert highlights |
| `NEON_PINK` | `#008B8B` | Alias for accent in hacker profile |
| `NEON_PURPLE` | `#006B6B` | Darker accent variant |

To change the palette, edit `lib/colors.sh` and update the hex values. The same values propagate to:

- `dotfiles/tmux/tmux.conf` (status bar colors)
- `dotfiles/sddm/themes/neon-minimal/theme.conf` (login screen)
- `dotfiles/i3/config` (bar and window colors)
- `lib/hacker_profile` (zsh prompt and terminal colors)

## SDDM Theme

The NEON MINIMAL SDDM theme lives in:

```
dotfiles/sddm/themes/neon-minimal/
├── theme.conf            # Theme configuration (colors, fonts)
├── metadata.desktop      # Theme metadata
└── Main.qml              # QML login interface
```

To customize the login screen:

1. Edit `dotfiles/sddm/themes/neon-minimal/theme.conf`
2. Re-run phase 0: `sudo ./setup_i3_kali.sh --phase 0`

## i3 Config

The i3 configuration is deployed to `~/.config/i3/config`. Key bindings:

| Key | Action |
|-----|--------|
| `Super+Enter` | Open terminal |
| `Super+d` | Application launcher (rofi) |
| `Super+Shift+q` | Close focused window |
| `Super+Arrow keys` | Navigate between workspaces |
| `Super+Shift+Arrow keys` | Move windows |
| `Super+f` | Fullscreen |
| `Super+Shift+r` | Reload i3 config |
| `Super+Shift+e` | Exit i3 |

To customize keybindings, edit `dotfiles/i3/config` before running the script, or edit `~/.config/i3/config` after installation.

## tmux Config

tmux is configured with:

- **Prefix:** `Ctrl+A` (remapped from `Ctrl+B`)
- **Split panes:** `|` (horizontal), `-` (vertical)
- **Navigate:** `Alt+Arrow keys`
- **Status bar:** NEON MINIMAL theme with session, window, and pane info

Config location: `~/.config/tmux/tmux.conf`

## Zsh Customization

### Oh-My-Zsh Plugins

Edit `~/.zshrc` to add/remove plugins:

```bash
plugins=(git docker kubectl ansible)
```

### Hacker Profile

The hacker profile at `~/.config/zsh/hacker_profile.zsh` can be extended with custom aliases:

```bash
# Add your custom aliases
alias my-tool='path/to/tool'
alias recon='nmap -T4 -A -v'
```

### Powerlevel10k

Reconfigure the Powerlevel10k prompt:

```bash
p10k configure
```

## Adding New Security Tools

To add a tool to the security suite:

1. Add the apt package name to `step_install_security_suite()` in `setup_i3_kali.sh`
2. Add aliases to `step_deploy_hacker_profile()` in the appropriate category
3. Add verification to `step_install_hexstrike_ai()` if needed

## Adding New Phases

To add a new phase:

1. Create a `step_*()` function in `setup_i3_kali.sh`
2. Add it to the `PHASES` array with a descriptive name
3. Add corresponding tests in `tests/test-phase-N.sh`
4. Update the phases documentation in `docs/phases.md`

| openCode config | `~/.config/opencode/.opencode.json` | Generated by script |

## Reverting Changes

To remove the NEON MINIMAL setup:

1. Remove installed packages manually (the script doesn't track uninstalls)
2. Delete generated configs: `rm -rf ~/.config/i3 ~/.config/tmux ~/.config/zsh`
3. Remove the SDDM theme: `sudo rm -rf /usr/share/sddm/themes/neon-minimal`
4. Reinstall your previous shell: `sudo chsh -s /bin/bash $(whoami)`
5. Remove checkpoint: `rm -rf ~/.cache/kali-i3/`
